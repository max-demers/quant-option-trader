# Automated Trading Engine

## Overview
The `trading_bot` module is the live execution arm of the project. It leverages the mathematical models developed in the `research` phase to actively scan the market, identify mispriced derivative contracts, and manage a portfolio autonomously.

## System Architecture

- **`trader.py`**: The central orchestrator. It uses a scheduling system to manage the bot's lifecycle, including periodic model recalibration, market scanning, and portfolio reconciliation.
- **`bates_model.py`**: The production-ready implementation of the Bates pricing engine. It encapsulates the calibration logic and robust pricing functions required for live analysis.
- **`market_data_handler.py`**: The data ingestion layer. Interfaces with the `yfinance` API to fetch real-time quotes, options chains, and risk-free rates. It is optimized to filter for liquidity and specific expiries.
- **`portfolio_manager.py`**: The risk and state management layer. It tracks available cash, open long/short option positions, and underlying stock holdings. It calculates unrealized PnL and logs all transactions locally.
- **`config.py`**: Centralized configuration file defining trading parameters, scheduler frequencies, and optimization bounds.

## Trading Strategy & Risk Management
1. **Statistical Arbitrage**: The bot compares live market mid-prices against the theoretical prices generated by the calibrated Bates model. It seeks to buy undervalued options and short overvalued ones based on a defined RMSE threshold.
2. **Delta Hedging**: To isolate the volatility premium and mitigate directional market risk, the portfolio manager calculates the aggregate Delta of the options portfolio and executes offsetting trades in the underlying asset (e.g., buying/selling shares of SPY).
3. **Automated Liquidation**: The system continuously evaluates open positions, automatically closing them if the market price converges to the theoretical model price or if the contract nears expiration to avoid assignment risk.
